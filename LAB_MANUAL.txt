PRODUCT RECOMMENDATION ENGINE - LABORATORY MANUAL

====================================================================
1) PROBLEM STATEMENT
====================================================================

Many teams struggle with product recommendation workflows because:
- User requests arrive in different formats (text, voice, image).
- Manual product matching is slow and inconsistent.
- Recommendations are often not grounded in a trusted catalog.

This application solves those problems by providing a single assistant that:
- Interprets customer needs from text, optional voice, and optional image input.
- Retrieves recommendations from catalog-backed search.
- Produces explainable guidance and value-oriented product suggestions.


====================================================================
2) WHAT THIS APPLICATION DOES
====================================================================

This application recommends products using:
- Text input (example: "I need affordable products for kitchen cleaning")
- Optional voice input
- Optional image upload

The application includes 3 tabs:
1. Customer Assistant
	- Ask for product recommendations.
2. Admin Catalog
	- View catalog products.
	- Upload a CSV or JSON catalog.
	- Add products manually.
	- Sync catalog to Azure AI Search.
3. Architecture
	- Shows technical flow and environment variable reference.

Important notes:
- Recommendations require Azure AI Search to be configured.
- Azure OpenAI is recommended, but not mandatory.
- Image and voice features only work when Azure Vision and Azure Speech are configured.

5) ONE-TIME SETUP
====================================================================

Step 1: Open PowerShell in the project folder
- In File Explorer, open the project folder.
- Right-click empty space and select "Open in Terminal".
- Confirm folder path ends with recommendation-engine.

Step 2: Verify Python installation
Run:

python --version

If you get a version (for example Python 3.11.x), continue.
If Python is not found:
- Install Python from https://www.python.org/downloads/
- During install, check "Add Python to PATH"
- Close and reopen terminal, then run python --version again

Step 3: Create a virtual environment (only if .venv does not exist)
Run:

python -m venv .venv

Step 4: Activate the virtual environment
Run:

.\.venv\Scripts\Activate.ps1

If script execution is blocked, run this once in PowerShell:

Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

Then activate again:

.\.venv\Scripts\Activate.ps1

Step 5: Install required packages
Run:

.\.venv\Scripts\python -m pip install -r requirements.txt

6) CONFIGURE ENVIRONMENT SETTINGS (.env)
====================================================================

You can configure the environment in two ways:

A) Minimum setup (required for recommendations)
	Required:
	- AZURE_SEARCH_ENDPOINT
	- AZURE_SEARCH_API_KEY
	- AZURE_SEARCH_INDEX_NAME

B) Full setup (best experience)
	Add all from option A, plus:
	- AZURE_OPENAI_ENDPOINT
	- AZURE_OPENAI_API_KEY
	- AZURE_OPENAI_DEPLOYMENT
	- AZURE_OPENAI_API_VERSION
	- AZURE_OPENAI_EMBEDDING_DEPLOYMENT
	- AZURE_VISION_ENDPOINT
	- AZURE_VISION_KEY
	- AZURE_SPEECH_KEY
	- AZURE_SPEECH_REGION
	- AZURE_DOCUMENT_INTELLIGENCE_ENDPOINT
	- AZURE_DOCUMENT_INTELLIGENCE_KEY

How to edit .env:
1. Open .env in VS Code.
2. Update the values copied from your Azure portal.
3. Save the file.

Example from template:

AZURE_SEARCH_ENDPOINT=https://your-search-service.search.windows.net
AZURE_SEARCH_API_KEY=your-key-here
AZURE_SEARCH_INDEX_NAME=product-catalog

Notes:
- Keep keys private. Do not share screenshots of your .env file.
- No quotes are needed around values.
- Do not add spaces before or after '='.

9) CATALOG FILE FORMAT (FOR ADMIN USERS)
====================================================================

Accepted upload formats:
- CSV
- JSON

Typical CSV headers:
- Category
- Product Name
- Description
- Price
- Tags
- Use Cases
- Benefits
- Image Hints

Data quality guidelines:
- Use clear product names.
- Add meaningful tags (for example: cordless, HEPA, pet-hair).
- Keep descriptions practical and specific.
- Add use cases and benefits as comma-separated values.

====================================================================
11) TROUBLESHOOTING
====================================================================

Problem: "No recommendations" and message about Azure datastore unavailable
Cause:
- Azure AI Search not configured or unreachable.
Fix:
1. Check AZURE_SEARCH_ENDPOINT, AZURE_SEARCH_API_KEY, AZURE_SEARCH_INDEX_NAME in .env.
2. In Admin Catalog tab, click "Sync catalog to Azure AI Search".
3. Retry your query.

Problem: App does not start, command not found
Cause:
- Virtual environment not activated or dependencies missing.
Fix:
1. Activate .venv again.
2. Reinstall dependencies:
	.\.venv\Scripts\python -m pip install -r requirements.txt

Problem: Voice input not working
Cause:
- Streamlit version or Azure Speech settings are missing.
Fix:
1. Confirm dependencies are installed from requirements.txt.
2. Confirm AZURE_SPEECH_KEY and AZURE_SPEECH_REGION in .env.

Problem: Image upload works but no image-based matching
Cause:
- Azure Vision is not configured.
Fix:
1. Set AZURE_VISION_ENDPOINT and AZURE_VISION_KEY in .env.
2. Restart app and retry.

Problem: Catalog uploaded but search results are old
Cause:
- New local catalog not synced to Azure AI Search.
Fix:
1. Go to Admin Catalog.
2. Click "Sync catalog to Azure AI Search" after every import/update.

Problem: Browser page does not open automatically
Fix:
- Manually open http://localhost:8501
